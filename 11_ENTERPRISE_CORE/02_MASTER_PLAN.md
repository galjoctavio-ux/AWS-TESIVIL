# 02. Master Plan

## Vision del Proyecto
Crear una plataforma educativa y operativa para Windows que, bajo un marco de calidad adaptado (ISO 9001), enseñe al usuario a pensar y orquestar a una IA avanzada (AntiGravity) para construir MVPs de software estables, excluyendo explícitamente dominios de alto riesgo.

## Alcance del MVP: El "Director de Orquesta IA"

### Objetivos Clave
1.  **Educación Activa**: El software explica *por qué* se pide cada dato (e.g., "Necesito los riesgos para que la IA no alucine soluciones peligrosas").
2.  **Generación de MVP**: El output final es un plan de ejecución tan detallado que AntiGravity puede codificar el MVP con mínima fricción.
3.  **Safety First**: Implementación de "Guardrails" metodológicos que desincentiven el uso en áreas legal/médica/casinos.

## Tecnología Sugerida (v1.5)
*   **UI**: WPF (Recomendado) para control total sobre ventanas, diálogos modales y experiencia de escritorio robusta.
*   **Motor Lógico**: Máquina de estados con validación de "Completitud" (No pasas de fase si tus riesgos no están mitigados).
*   **Integración IA**: Generación de prompts en formato Markdown enriquecido, optimizados específicamente para el contexto de AntiGravity.

## Fases de Ejecución Refinadas

### Fase 1: Arquitectura Pedagógica (Actual)
*   Diseño de los mensajes explicativos (Tooltips educativos).
*   Definición de los "Guardrails" éticos en el flujo de Ideación.

### Fase 2: Prototipado "Teacher-Led"
*   UX que se siente como un tutor experto guiando la mano del usuario.
*   Validación de que el usuario entiende que EL es el director, y la IA es el obrero.

### Fase 3: Conector AntiGravity
*   Creación de la librería de prompts maestros que traducen las decisiones del usuario en instrucciones técnicas para AntiGravity.

### Fase 4: Validación de MVP
*   Prueba de concepto: Generar una app sencilla (e.g., ToDo List) usando exclusivamente el flujo del software + AntiGravity.
